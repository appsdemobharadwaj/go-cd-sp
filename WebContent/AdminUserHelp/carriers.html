Carriers

The Carriers page allows setup of carrier details such as Carrier code, printer models, etc. It also provides the link to the Ship Method Mapping page which allows mapping of Ship methods to Carrier Service Levels into Shipping Page. To perform carrier configuration, first select the Location and Carrier Name and click on the ‘Go’ button. For a carrier to be active in ShipConsole (i.e. for its ship methods to be available in the shipments page, the ‘Enable Flag’ check box has to be checked for that carrier).

Figure 5.1


1. Carrier Server Details 

I. Carrier Code: We are using carrier code for internal validations and mapping such as UPS, FDXE, FDXG and others

II. Enable flag: Used in order to activate/deactivate particular carrier.

Note: We need to check this check box in order to go to ship.

III. Protocol: Specify the Protocol for Carrier such as http/https.

IV. Port: port number of the carrier.

V. Mode: This is used to know the mode of transmission such as it is atom/web service/connect ship etc.

VI. Server IP Address: IP address where carrier API located.

VII. User name: User name with which we can login to carrier site.

VIII. Password: Password with which we can login to carrier site.

IX. Prefix: Prefix that is given by the carrier.

X. Domain: Specify the domain for carrier such as GLOG( is applicable only for LTL’s).

XI. Account Number : This is the attribute value which defaults the carrier account number in Shipments page.

2. Printer Details:

I. Label Format: The format in which label should be printed.

II. LabelStock Type: Label size.

III. DocTab Location: Peelable portion of label. Choose NA if label stock does not have a doc tab.


3. Email Notification Details:

       ShipConsole leverages carrier’s notification system to inform carrier to send email notifications with tracking numbers to the addresses provided to that carrier.        This section describes some setups for the notifications to work. Please also check the setups for email notifications in the ShipConsole Install Guide.


Email Notification Required?: Check this to activate email notifications for selected carrier.

Ship Notification: Select if you want the Ship notification to be sent by the carrier.

Reference1 Required: Select if you want to send Reference1 value (selected in ShipConsole Profile Options page) in the email notification.

Reference2 Required: Select if you want to send Reference2 value (selected in ShipConsole Profile Options page) in the email notification

Exception Notification: Select if you want the notification to be sent by carrier if there is an exception in delivering the package.

Format: E-Mail Notification format

Customer Name: Select if you want the notification to be sent by the carrier, Customer Name

Ship Method Mapping:


“Go To Ship Method Mapping” button is enabled when you click on “Go” button after selecting Location and Carrier Name.


                                               Figure 5.7


Click on the “Go to Ship Method Mapping” button to map Ship Methods to UPS Service Levels. This screen includes User Defined Ship Method (Display name in Shipments Page), Carrier Service Level, Validation, Enabled and International Flag.

When we navigate into this screen, Location and Carrier Names will be carried over from Carrier page. Based on these combinations, all defined ship methods will be loaded into page.


                                                               Figure 5.8


1. User Defined Ship Method: The name that is displayed for that ship method in the shipments page.


2. Carrier Service Level: This field contains different services provided by carriers. Select appropriate service level for that ship method from the list of values.


3. Validation: This field represents the different validations that carriers provide during shipping.

For following service levels, the SATURDAY SHIPMENT FLAG ENABLE validation should be selected. For other UPS service levels, select NO VALIDATION REQUIRED.

Carrier Service level Validation

UPS 2nd Day Air SATURDAY SHIPMENT FLAG ENABLE

UPS Next Day Air SATURDAY SHIPMENT FLAG ENABLE

UPS Next Day Air Early A.M SATURDAY SHIPMENT FLAG ENABLE


4. Enabled: To enable/disable that ship method in the shipments page. Disabled ship methods are not displayed.


5. International: If the service level is for international shipments.
